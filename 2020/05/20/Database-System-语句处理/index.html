语法分析与翻译 parsing and translation

执行 evolution

优化 optimization

因为不同的语句可能得到相同的结果，我们要执行当中代价最小、效率最高的那种；同一个语句还可以有很多种不同的实现算法（重点）。

查询执行计划：不仅要标注关系代数关系，还要标注算法。

优化器的功能：选出代价最小的等价语句。代价小指需要的时间，需要估算。

时间包括：disk accesses + CPU + network communication，我们主要估算磁盘IO时间。IO时间包括寻道时间、读时间、写时间。为了简便，我们不区分读写，统一成传输时间。Tt表示传输单位时间，Ts表示寻道单位时间。这样IO时间只需要计算传输时间和寻道时间。

缓存很大程度上会影响IO时间，因为如果缓存里面有就没必要请求磁盘了，但这个是动态的，我们只考虑最小缓存情况和最好缓存情况。

选择算法

等值

  1. A1 顺序，不用索引。只要一次寻道，从头到尾，线性搜索。设br是数据所占的块数，最坏的一共的时间是一次寻道时间+br次IO时间（传输时间）。如果是按照一个key来查询的话，因为是唯一的所以不必全部读，需要一次寻道+br/2次传输。

    好处是不管什么情况下都可能用，无论条件、顺序、索引。但因为原来的数据不一定排序了，所以二分法搜索不太常用；如果排序了还可以用。另外二分法会导致寻道次数增加。

  2. A2 候选码索引辅助的相等条件的扫描 index scan。默认是B+树，搜索条件建立在索引项上

    。

    HI：根节点到叶子结点的过程，然后把所在block取回。

  3. A3 非候选码的主索引上的相等条件 scan。可能返回多条记录。

    前面HI还是寻找的过程，后面因为是主索引的顺序所以只需要一次寻道定位，之后要读所有的块。b是大约所有符合条件的记录占的块数。

  4. A4 二级索引的等值比较。第一种这个二级索引也是key，不会重复，这时开销和2是一样的。但假如不是一个key，就涉及到多次的寻道，因为信息不是按照二级索引排序的。

涉及比较的

  1. 遍历
  2. A5 基于主索引的比较：假设是按照主索引来排序和选择的，如果是>=，就定位到第一条符合的，然后一直扫描下去；如果是<=，就从开头一直读到不符合，甚至不用索引。
  3. A6 基于辅助索引的比较：在叶子节点上使用2的方法，但是这样开销非常大。

涉及逻辑操作and or not 的

  1. A7 用一个索引：在所有单个条件里面选一个条件最小的来计算读到内存，然后在内存里再验证是否符合其他条件（and）。
  2. A8 用组合索引（composite index / multiple-key index）：建立在我所需要的属性上，可以用A3-A5算法。
  3. A9 先把所有的条件中涉及到的元组地址取回来，然后取交集，在用这些地址稍微排个序在取本体（and）。
  4. A10 在A9的情况下取交集（or）。
  5. not的计算：1. 线性扫描 2. 转换成肯定句

排序

我们的重点是，内存不够大时的外部归并排序。

M:只有三块内存，一块内存只能放一条元组。第一步先三个三个读进来排序（Pass0），然后马上写成文件放出去。因为只能同时最多打开三个文件来读写，读两个run file，写一个。写的时候是分别读一条进来，写一个小的，然后再从小的原来所在的文件读一条，直到结束。

产生的文件数=本轮文件数 /（最多打开文件数-1）取上整。如果内存很大（大于N）一次归并就够了。

代价分析：

  1. 传输开销

$$
总共轮数是 \lceil log_{M-1} (b_r/M)\rceil,不包括Pass0；\
也就是\lceil log_{M-1}(N_0)\rceil。N_0就是Pass0产生的文件数。\
块传输的话，我们不计算最后一次的写的时间。因为有可能是不写回磁盘而是传给上一步操作。\
每一次块传输的次数都是2b_r.\
整个外部排序的block\quad transfer数目是轮数\times每轮的传输数-最后一次，也就是\
BTcost = 2 b_r (\lceil log_{M-1}(b_r/M)\rceil+1)-b_r.
$$

  1. 寻道开销
    $$
    第一次run的时候，要产生一个run\quad file，\就需要先定位到原来的M-1个文件开始，再定位到接收答案的文件开始，所以需要两次寻道。\这样Pass0一共的寻道次数就是2\lceil b_r/M\rceil次。\那在排序过程中不可能内存真的小到每次只能放一块，所以引入一个b_b的概念\表示一次可以读b_b个块。一共是b_r块，就需要b_r/b_b次读块。\每一个块在前一个Pass中被写，后一个Pass中被读，\所以是2\lceil b_r/b_b \rceil 次的寻道（因为他们就不像最开始的记录那样挨着）\但最后一次不用算，所以一共的次数是\2\lceil b_r/M\rceil+2\lceil b_r/b_b\rceil(\lceil log_{M-1}(b_r/M)\rceil)-b_r/b_b.
    $$
